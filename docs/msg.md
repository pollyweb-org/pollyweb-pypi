# `Msg` ‚Äî PollyWeb Message Envelope

`pollyweb.msg` implements the PollyWeb message envelope (`pollyweb.org/MSG:1.0`): a signed, tamper-evident JSON structure that domains exchange to communicate securely.

---

## Quick start

### Sending via a `Domain` (recommended)

```python
import pollyweb as pw

msg = pw.Msg(To="receiver.dom", Subject="Hello@Host", Body={"text": "hi"})
domain = pw.Domain(Name="sender.dom", PrivateKey=private_key, DKIM="pw1")
signed = domain.sign(msg)
```

### Direct signing

```python
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey
import pollyweb as pw

private_key = Ed25519PrivateKey.generate()
msg = pw.Msg(From="sender.dom", To="receiver.dom", Subject="Hello@Host", DKIM="pw1", Body={"text": "hi"})
signed = msg.sign(private_key)

# Receiver: validate
public_key = private_key.public_key()
signed.validate(public_key)  # True, or raises pw.MsgValidationError
```

## `Msg` fields

`Msg` is a **frozen dataclass** ‚Äî all fields are immutable after construction. Methods that modify state return a new instance.

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `To` | `str` | ‚úÖ | ‚Äî | Receiver's domain name. |
| `Subject` | `str` | ‚úÖ | ‚Äî | Method to invoke on the receiver (e.g. `Hello@Host`). |
| `From` | `str` | ‚Äî | `""` | Sender's domain name. Set by `Domain.sign()` if omitted. |
| `DKIM` | `str` | ‚Äî | `""` | Name of the sender's public key in DNS (e.g. `pw1` ‚Üí `pw1._domainkey.pw.sender.dom`). Set by `Domain.sign()` if omitted. |
| `Body` | `dict` | ‚Äî | `{}` | Arbitrary JSON-serialisable payload. |
| `Correlation` | `str` | ‚Äî | UUID4 | Unique message ID generated by the sender, used for deduplication. |
| `Timestamp` | `str` | ‚Äî | UTC now | ISO-8601 timestamp with millisecond precision (e.g. `2025-06-01T12:00:00.000Z`). |
| `Schema` | `str` | ‚Äî | `pollyweb.org/MSG:1.0` | Wire-format schema version. |
| `Hash` | `str \| None` | ‚Äî | `None` | SHA-256 hex digest of the canonical form. Set by `sign()`. |
| `Signature` | `str \| None` | ‚Äî | `None` | Base64-encoded Ed25519 signature. Set by `sign()`. |

`validate()` requires `From`, `To`, `Subject`, `Correlation`, `Timestamp`, and `DKIM` to be non-empty, so messages must have those fields populated (directly or via `Domain.sign()`) before validation.

## Methods

### `msg.canonical() ‚Üí bytes`

Returns the canonical JSON bytes used for hashing and signing.

The canonical form contains only `Schema`, `Header`, and `Body` ‚Äî never `Hash` or `Signature`. Keys are sorted and serialised with no whitespace (JCS, RFC 8785).

```python
msg.canonical()
# b'{"Body":{"text":"hi"},"Header":{"Correlation":"...","DKIM":"pw1","From":"sender.dom",...}}'
```

### `msg.sign(private_key: Ed25519PrivateKey) ‚Üí Msg`

Returns a **new** `Msg` with `Hash` and `Signature` set. The original is unchanged.

```python
signed = msg.sign(private_key)
assert msg.Hash is None       # original untouched
assert signed.Hash is not None
```

Internally: `SHA-256(canonical())` ‚Üí `Hash`; `Ed25519.sign(canonical())` ‚Üí base64 ‚Üí `Signature`.

### `msg.validate(public_key: Ed25519PublicKey | None = None) ‚Üí bool`

Validates structure and signature. Returns `True` on success, raises `MsgValidationError` otherwise.

`public_key` is optional. When omitted, the key is fetched from DNS using the
message's `DKIM` selector and `From` domain:

```
{DKIM}._domainkey.pw.{From}  (DNS TXT record)
```

The TXT record must follow the standard DKIM wire format:
`v=DKIM1; k=ed25519; p=<base64-encoded public key>`

**DNSSEC is required** ‚Äî the DNS response must have the Authenticated Data (AD)
flag set. If the domain has not enabled DNSSEC, validation fails with
`MsgValidationError("DNSSEC not enabled for ‚Ä¶")`.

Checks performed in order:

1. `Schema` matches `pollyweb.org/MSG:1.0`
2. All required header fields (`From`, `To`, `Subject`, `Correlation`, `Timestamp`, `DKIM`) are non-empty
3. `Hash` and `Signature` are present
4. If no `public_key` supplied: resolve from DNS (DNSSEC required)
5. `SHA-256(canonical())` matches stored `Hash`
6. Ed25519 signature verifies against `canonical()`

```python
# Explicit key (no DNS lookup)
signed.validate(public_key)

# Key resolved from DNS (requires DNSSEC on sender.dom)
signed.validate()
```

### `msg.to_dict() ‚Üí dict`

Serialises to a Python dict matching the wire format (ready for `json.dumps`).

```python
{
    "Header": {
        "From": "sender.dom",
        "To": "receiver.dom",
        "Subject": "Hello@Host",
        "Correlation": "3fa85f64-...",
        "Timestamp": "2025-06-01T12:00:00.000Z",
        "DKIM": "pw1",
        "Schema": "pollyweb.org/MSG:1.0"
    },
    "Body": {"text": "hi"},
    "Hash": "ee6ca2a4...",
    "Signature": "Lw7sQp6z..."
}
```

`Hash` and `Signature` are omitted if `None` (i.e. unsigned messages).

### `Msg.from_dict(d: dict) ‚Üí Msg`

Deserialises from a dict. Inverse of `to_dict()`.

```python
msg = Msg.from_dict(json.loads(raw_json))
```

```yaml
  Schema: pollyweb.org/MSG:1.0

  Subject: Hello@Host

Body: { ... }

Hash: ee6ca2a43ec05d...

```

## Error handling

`MsgValidationError` is raised by `validate()` with a descriptive message:

| Message | Cause |
|---|---|
| `Unsupported schema: X` | `Schema` field is not `pollyweb.org/MSG:1.0` |
| `Missing From` / `To` / ‚Ä¶ | A required header field is empty |
| `Missing Hash` | `Hash` is `None` (message not signed) |
| `Missing Signature` | `Signature` is `None` |
| `Hash mismatch` | Body or header was tampered with after signing |
| `Invalid signature` | Signature does not verify (wrong key or tampered content) |
| `Malformed signature: ‚Ä¶` | `Signature` is not valid base64 |

## Design notes

**Immutability** ‚Äî `Msg` is a frozen dataclass. `sign()` returns a new instance rather than mutating in place, making it safe to pass unsigned messages around without risk of accidental modification.

**Ed25519 over RSA** ‚Äî Ed25519 (RFC 8032 / RFC 8463) provides a ~128-bit security level with 64-byte signatures, is immune to timing attacks by design, and requires no padding-scheme decision. RSA-2048 with PKCS1v15 (the DKIM default) is deprecated in FIPS 186-5.

**JCS canonicalisation** ‚Äî Keys are sorted and whitespace is removed before hashing, following RFC 8785. This guarantees the same bytes regardless of how the dict was constructed.

**`Header` as a wire concept only** ‚Äî In Python, all fields sit flat on `Msg`. The `Header` object only appears inside `to_dict()` / `from_dict()` to match the JSON wire format.

**CamelCase fields** ‚Äî Field names on `Msg` match the wire-format names (e.g. `From`, `To`, `Subject`) for consistency between the Python API and the JSON representation.

**`From` and `DKIM` are optional at construction** ‚Äî They default to `""` so that a `Msg` can be built without knowing the sender. `Domain.sign()` fills them in automatically. `validate()` still requires them to be non-empty.
